# 关键字static

## 背景

创建类时，就是在描述这个类对象的外观与行为。除非用new创建那个类的对象，否则，实际上并未获得任何对象。执行new来创建对象时，数据存储空间才被分配，其方法才被外界调用。
   有两种情况，通过new对象的方法无法解决。

   - 只想为 **某个特定域（成员变量）** 分配单一存储空间，而不去考虑究竟要创建多少对象，甚至就根本不创建任何对象；
   - 希望 **某个方法** 不与包含它的类的任何对象关联在一起。也就是说，即使没有创建对象也可以调用方法；

## 特点
- 被 static 修饰的成员属于类，不属于类的某个对象；
- static 修饰的成员被多个对象共享；
- static 修饰的成员属于类，但是影响每一个对象；

## 修饰对象
- 成员变量
    eg. static int a = 10;
- 成员方法
    eg. static void 方法名(){}
- 代码块
    eg. static{静态代码块，优先于主方法执行}

## 成员变量和成员方法的使用
- 通过类名调用。格式：类名.被static修饰的成员变量或方法。
    - DemoClass.variable = 2;
    - DemoClass.getName();

- 通过对象名调用。和一般的创建对象调用成员变量和成员方法一样。

## 静态代码块

**静态代码块** 是指定义在成员位置，使用static修饰的代码块。

>特点

- 优先于 **主方法和构造代码块** 执行，当以任意形式使用到该类时执行；（任意形式：创建对象、静态调用等）
- 该类不管创建多少个对象，静态代码块只执行一次；
- 用于给静态变量赋值，对类进行初始化；
